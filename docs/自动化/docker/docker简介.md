### 虚拟化说明
虚拟化主要是对内核来讲的，用户层不感知虚拟化，而内核需要感知，因为CPU有0-3四个特权等级，分别对应不同权限，内核态运行在0，用户态运行在3，对于直接
运行于硬件的内核来讲，他的权限就是0，而对于运行于虚拟机的内核来讲，作为内核，他的权限应该是0，但是他又是直接运行于硬件的内核上的一个普通用户态程序，
所以他的实际权限是3，这样运行中可能就有一些问题，而VirtualBox和Vmware就是解决这个问题的，他们会虚拟化出一层硬件层，然后告诉虚拟机中的内核可以直
接操作，相当于0权限，但是实际上虚拟化层接到请求后会将其转发到真正的物理机的内核，由其进行处理；这样处理可以解决特权等级问题，但是又有新问题，就是
虚拟机的每次调用都经过虚拟化层的转发，效率比较低，后来又有了解决方案，因为0-3这四个特权级别之前设计的时候没考虑虚拟化，直接用了0和3，中间的相当于
浪费了，所以硬件后来又加了一个新的标志位，表示是在虚拟机内核下还是真正的物理机内核下，这样虚拟机内核执行指令就不用经过虚拟化层转发了，而是直接调用
硬件；
